# Estrategia de capturas

Las capturas deben mostrar el flujo real de la demo publica, sin credenciales visibles ni datos tecnicos internos.

## Capturas requeridas

| Archivo | Vista | Criterio |
|---|---|---|
| `login.png` | Login | Inputs vacios, usuarios demo visibles, contrasenas como `********` |
| `dashboard.png` | Dashboard | Tarjetas de resumen y alertas operativas |
| `inventario.png` | Inventario | Stock, lotes o alertas de vencimiento |
| `productos.png` | Productos | Catalogo, categorias, precios y estado |
| `ventas-pos.png` | POS / Ventas | Busqueda de productos y carrito |
| `caja.png` | Caja | Estado de caja, movimientos o cierre |
| `reportes.png` | Reportes | Indicadores comerciales y tablas |

## Recomendaciones

- Usar navegador en 1440x900 o 1366x768.
- Capturar con la demo desplegada, no con `localhost`.
- No mostrar tokens, herramientas de desarrollo ni URLs internas.
- Usar datos seed de ejemplo.
- Guardar PNG optimizados en esta carpeta.
